# 从零启动当前项目

本文档说明如何在一台全新的电脑上，从零把当前 Web3 Todo DApp 跑起来。内容包括环境准备、依赖安装、本地链启动、合约部署、前端启动、MetaMask 配置，以及常见问题排查。

## 1. 项目包含什么

这个项目是一个典型的 Web3 全栈应用，包含以下部分：

- 智能合约：`contracts/Todo.sol`
- 部署脚本：`scripts/deploy.js`
- 合约测试：`test/Todo.test.js`
- 本地链：Hardhat Network
- 前端：`frontend/` 下的 React + Vite 应用
- 钱包连接：MetaMask
- 合约调用：ethers.js

它的核心流程是：

1. 启动本地区块链。
2. 将 Todo 合约部署到本地链。
3. 启动前端页面。
4. 在 MetaMask 中切换到本地网络并连接钱包。
5. 通过页面发起链上交易，完成增删改查。

## 2. 启动前需要准备什么

### 2.1 必备软件

你需要先安装以下软件：

- Node.js（建议 18 以上，最好是当前稳定版）
- npm（随 Node.js 一起安装）
- Git
- MetaMask 浏览器插件
- 一个现代浏览器，例如 Chrome、Edge 或 Brave

### 2.2 建议的操作习惯

- 不要在主网环境直接测试这个项目。
- 优先使用本地 Hardhat 网络 `31337`。
- 测试账户只用于开发，不要导入真实资产钱包。

## 3. 克隆项目

如果你还没有项目代码，可以先执行：

```bash
git clone https://github.com/iYangJike/web3-todo-dapp.git
cd web3-todo-dapp
```

如果你已经在当前目录，就直接进入项目根目录即可。

## 4. 安装依赖

这个项目分成两个依赖层：

- 根目录：Hardhat、ethers、dotenv、openzeppelin 等
- 前端目录：React、Vite、ethers、Tailwind 相关依赖

### 4.1 安装根目录依赖

在项目根目录执行：

```bash
npm install
```

### 4.2 安装前端依赖

进入前端目录执行：

```bash
cd frontend
npm install
```

安装完成后你就具备了编译合约、启动本地链和运行前端的基础环境。

## 5. 启动本地 Hardhat 网络

回到项目根目录，执行：

```bash
cd /Users/a1/web3-todo-dapp
npx hardhat node
```

启动后你会看到：

- 本地 JSON-RPC 服务地址：`http://127.0.0.1:8545`
- 一组预置测试账户
- 每个账户都有大量测试 ETH

### 5.1 这一步的作用

Hardhat node 是本地开发链，它的作用是：

- 模拟真实区块链环境
- 提供测试账户和测试币
- 让你无需真实资金就能发交易
- 方便调试合约逻辑和前端交互

### 5.2 重要提醒

- 这个终端不要关掉。
- 如果关闭，本地链就会停止。
- 重启后链上状态通常会重置，需要重新部署合约。

## 6. 部署智能合约到本地链

保持本地 Hardhat 节点运行，打开另一个终端，在项目根目录执行：

```bash
npx hardhat run scripts/deploy.js --network localhost
```

部署成功后会打印合约地址，例如：

```text
Todo 合约已部署到: 0x5FbDB2315678afecb367f032d93F642f64180aa3
```

### 6.1 这一步的作用

部署脚本会：

- 获取部署账户
- 编译并部署 `Todo` 合约
- 输出部署后的合约地址

这个地址是前端连接链上合约的关键参数。

### 6.2 当前项目的合约地址

目前前端中已经配置了一个本地部署地址，位于：

- `frontend/src/hooks/useContract.js`

如果你重新启动本地链并重新部署，合约地址可能变化，需要同步更新这个常量：

```js
const CONTRACT_ADDRESS = "你的新合约地址";
```

## 7. 启动前端项目

进入前端目录并启动开发服务器：

```bash
cd /Users/a1/web3-todo-dapp/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

启动成功后，通常会看到：

- 本地访问地址：`http://localhost:5173/`
- 局域网访问地址：`http://你的IP:5173/`

### 7.1 这一步的作用

前端负责：

- 连接 MetaMask
- 切换网络
- 实例化合约
- 读取 Todo 列表
- 发起创建、完成、删除交易

## 8. 配置 MetaMask

### 8.1 添加本地网络

打开 MetaMask，添加一个网络：

- 网络名称：Hardhat Local
- RPC URL：http://127.0.0.1:8545
- Chain ID：31337
- 货币符号：ETH
- 区块浏览器 URL：留空

### 8.2 导入测试账户

为了方便测试，你可以导入 Hardhat 提供的默认测试账户私钥。

导入方式见另一份文档：`导入 Hardhat 测试账户到 MetaMask.md`。

### 8.3 这一步的作用

MetaMask 是 DApp 的身份层和签名层：

- 账户地址代表用户身份
- 交易必须由钱包签名确认
- 钱包网络必须和本地链匹配

## 9. 运行时的正确顺序

建议按下面顺序启动：

1. 启动 Hardhat node。
2. 部署合约到 localhost。
3. 启动前端 Vite 服务。
4. 打开 MetaMask，切到 Hardhat Local。
5. 导入测试账户并连接钱包。
6. 开始创建、完成和删除 Todo。

## 10. 交互流程说明

当你在页面点击“连接钱包”时，前端会做这些事：

1. 检查浏览器中是否安装了 MetaMask。
2. 自动请求切换到 `31337` 网络。
3. 请求用户授权账户。
4. 检查当前地址是否有合约代码。
5. 创建合约实例并读取 Todo 列表。
6. 后续的增删改操作都通过钱包签名发交易。
7. 交易确认后刷新列表。

## 11. 常见问题排查

### 11.1 页面提示当前网络不对

可能原因：

- MetaMask 还在 Mainnet
- 没切到 Hardhat Local

解决方法：

- 在 MetaMask 中切换到 Hardhat Local
- 重新点击连接钱包

### 11.2 页面提示没有找到合约

可能原因：

- 你重新启动了本地链，但没有重新部署合约
- 前端的 `CONTRACT_ADDRESS` 还是旧地址

解决方法：

- 重新运行部署脚本
- 把新地址更新到 `frontend/src/hooks/useContract.js`

### 11.3 添加 Todo 时失败

可能原因：

- 没连接钱包
- 还没切到本地链
- 交易还未确认

解决方法：

- 先连接钱包
- 检查账户是否有测试 ETH
- 等待交易确认后再刷新

### 11.4 MetaMask 不弹出切网确认

可能原因：

- 浏览器屏蔽了弹窗
- MetaMask 插件状态异常

解决方法：

- 检查浏览器插件权限
- 刷新页面后重新连接

## 12. 一键检查清单

启动前你可以检查下面这些项是否都已完成：

- Node.js 已安装
- 根目录依赖已安装
- 前端依赖已安装
- Hardhat node 已启动
- 合约已部署到 localhost
- 前端 dev server 已启动
- MetaMask 已安装
- MetaMask 已添加 Hardhat Local 网络
- 已导入测试账户
- 页面已连接钱包

## 13. 总结

这个项目从零启动的关键就是三件事：

- 本地链要先起来
- 合约要先部署
- 前端和 MetaMask 必须连到同一条链

只要这三件事都正确，整个 Web3 Todo DApp 就可以完整运行起来。
