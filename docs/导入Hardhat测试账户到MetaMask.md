# 导入 Hardhat 测试账户到 MetaMask

本文档详细说明如何把 Hardhat 本地链提供的测试账户导入到 MetaMask。这样做的目的是让你可以在本地网络上直接签名交易、创建 Todo、切换状态和删除任务。

## 1. 先理解这是什么

Hardhat 在启动本地链时，会默认生成一组测试账户。

其中最常用的是第 0 个账户：

- 地址：`0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266`
- 私钥：`0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80`
- 测试余额：通常是 `10000 ETH`

这里要特别注意：

- 这不是主网资产
- 这不是代币合约 token
- 这是一个测试账户的私钥
- 只适合本地 Hardhat 网络开发

## 2. 为什么要导入这个账户

导入 Hardhat 测试账户的作用是：

- 让 MetaMask 拿到一个可以签名的本地账户
- 让你在页面里真正发交易
- 方便测试链上写操作：创建、完成、删除 Todo
- 能在本地环境看到测试 ETH 余额

如果不导入账户，你通常只能看页面，不能顺畅地测试交易流程。

## 3. 前置条件

在导入之前，你需要先完成这些事情：

1. 已经启动 Hardhat 本地链。
2. 已经在 MetaMask 中添加了 Hardhat Local 网络。
3. 你知道当前本地链的 RPC 地址是 `http://127.0.0.1:8545`。

如果本地链没启动，MetaMask 即使导入私钥，也无法正确和链通信。

## 4. 导入步骤

### 4.1 打开 MetaMask

在浏览器右上角点击 MetaMask 图标，打开钱包插件。

### 4.2 切换到 Hardhat Local 网络

在网络下拉菜单中选择你添加的本地网络：

- 网络名称：Hardhat Local
- Chain ID：31337

如果还没有这个网络，请先去添加网络。

### 4.3 打开“导入账户”

进入 MetaMask 后：

1. 点击右上角头像。
2. 选择“导入账户”或“Import Account”。
3. 选择“私钥”导入方式。

### 4.4 粘贴 Hardhat 默认私钥

把下面这串私钥粘贴进去：

```text
0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
```

然后点击导入。

### 4.5 检查导入结果

导入成功后，你应该能看到对应账户地址：

```text
0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266
```

并且在 Hardhat Local 网络下可以看到大量测试 ETH。

## 5. 导入后如何使用

导入成功后，你可以直接回到项目页面：

1. 刷新前端页面。
2. 点击“连接 MetaMask”。
3. MetaMask 会请求连接这个账户。
4. 连接成功后就可以创建、切换和删除 Todo。

## 6. 如果看不到余额怎么办

如果你导入后没有看到余额，先检查下面几项：

### 6.1 当前网络是否正确

必须在 Hardhat Local 网络下查看余额。

如果你在 Mainnet 或 Sepolia，看不到本地测试余额是正常的。

### 6.2 本地链是否还在运行

Hardhat 本地链必须保持启动状态。

如果你关闭了运行 `npx hardhat node` 的终端：

- 交易无法发送
- 页面可能无法读取数据
- MetaMask 仍然能显示账户，但链上状态会断开

### 6.3 账户是否导入正确

确认你导入的是私钥，不是助记词，也不是普通地址。

## 7. 为什么这个私钥可以公开看

Hardhat 默认测试账户的私钥是公开固定的，设计目的就是为了开发调试。

它之所以可以公开，是因为：

- 这些账户只用于本地开发网络
- 本地网络没有真实资金风险
- 每次启动都用于模拟测试环境

但是请注意：

- 绝对不要把真实钱包的私钥公开
- 绝对不要把主网助记词写进代码或聊天

## 8. 换电脑后怎么办

如果你换了一台电脑，Hardhat 默认测试账户一般不需要你额外备份，因为它是固定的开发账户。

你只需要：

1. 在新电脑启动 Hardhat node。
2. 在 MetaMask 添加 Hardhat Local 网络。
3. 再次导入同一把测试私钥。

这样就能重新得到同样的测试账户。

## 9. 导入账户的安全边界

这个账户的使用范围非常明确：

- 适合：本地开发、教学演示、功能测试
- 不适合：主网转账、真实资产管理、长期保存

只要你还在本地 Hardhat 环境里测试，它就是安全的；一旦进入真实网络，就必须换成自己的真实钱包并妥善备份。

## 10. 快速版步骤总结

如果你只想快速执行，按下面做就行：

1. 启动 Hardhat node。
2. 在 MetaMask 添加 Hardhat Local 网络。
3. 打开“导入账户”。
4. 粘贴私钥：

```text
0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
```

5. 导入后切回 Hardhat Local 网络。
6. 回到 DApp 页面连接钱包。
7. 开始测试 Todo 的链上操作。

## 11. 总结

Hardhat 测试账户的核心价值在于：

- 给你一个带测试币的本地钱包
- 让你能在无风险环境里练习签名与交易
- 让 Web3 DApp 的开发和演示流程更顺畅

只要记住它是“本地开发专用”，就可以放心使用。
